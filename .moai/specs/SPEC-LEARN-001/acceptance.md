# SPEC-LEARN-001 인수 기준

## 📋 인수 기준 개요

**목표**: 플립 카드 학습 시스템의 품질 검증
**검증 범위**: 학습 세션 → 플립 카드 → TTS 음성 → 네비게이션 → 완료
**검증 방법**: Given-When-Then 시나리오 기반 테스트

---

## 🎯 Phase 1: 핵심 학습 기능

### AC-1: 학습 세션 시작
**우선순위**: 높음 (필수)

**시나리오 1.1: 학습 시작 버튼 표시**
```gherkin
GIVEN 사용자가 5개의 단어를 포함한 단어 세트 상세 페이지에 있을 때
WHEN 페이지가 로드되면
THEN "학습 시작" 버튼이 표시된다
AND 버튼이 활성화 상태이다
```

**검증 방법**:
- E2E 테스트: `/wordsets/[id]` 페이지 접근 → 버튼 렌더링 확인
- 단위 테스트: WordSetDetail 컴포넌트 렌더링 테스트

**시나리오 1.2: 빈 세트 처리**
```gherkin
GIVEN 사용자가 단어가 없는 세트 상세 페이지에 있을 때
WHEN 페이지가 로드되면
THEN "학습 시작" 버튼이 비활성화 상태이다
AND "학습하려면 단어를 추가하세요" 안내 메시지가 표시된다
```

**검증 방법**:
- E2E 테스트: 빈 세트 페이지 접근 → 버튼 `disabled` 확인
- Accessibility 테스트: 비활성화 버튼 aria-disabled="true" 확인

**시나리오 1.3: 학습 모드 선택 화면 표시**
```gherkin
GIVEN 사용자가 단어 세트 상세 페이지에 있을 때
WHEN "학습 시작" 버튼을 클릭하면
THEN 학습 모드 선택 화면으로 이동한다
AND URL이 `/wordsets/[id]/learn`으로 변경된다
AND "순서대로 학습" 버튼이 표시된다
AND "랜덤 학습" 버튼이 표시된다
AND "뒤로" 버튼이 표시된다
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → URL 변경 → 모드 선택 화면 렌더링 확인
- 스크린샷 테스트: 모드 선택 화면 UI 비교

### AC-2: 학습 모드 선택
**우선순위**: 높음 (필수)

**시나리오 2.1: 순서대로 학습 모드**
```gherkin
GIVEN 사용자가 학습 모드 선택 화면에 있을 때
WHEN "순서대로 학습" 버튼을 클릭하면
THEN 첫 번째 단어의 앞면(뜻)이 표시된다
AND 진행률이 "1 / 5"로 표시된다
AND 진행률 바가 20% (1/5)로 표시된다
AND 세션 스토리지에 `mode: 'sequential'`이 저장된다
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → 첫 카드 렌더링 → 진행률 확인
- 세션 스토리지 테스트: `learn_session_${wordSetId}` 키 확인
- 단위 테스트: useLearnSession 훅 테스트 (순서 유지 확인)

**시나리오 2.2: 랜덤 학습 모드**
```gherkin
GIVEN 사용자가 학습 모드 선택 화면에 있을 때
WHEN "랜덤 학습" 버튼을 클릭하면
THEN 랜덤하게 셔플된 첫 번째 단어의 앞면(뜻)이 표시된다
AND 진행률이 "1 / 5"로 표시된다
AND 세션 스토리지에 `mode: 'random'`이 저장된다
AND 단어 순서가 원래 순서와 다르다 (셔플됨)
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → 첫 카드 렌더링 → 진행률 확인
- 단위 테스트: Fisher-Yates 셔플 알고리즘 테스트
- 확률 테스트: 10회 반복 시 최소 1회 이상 순서 변경 확인

**시나리오 2.3: 뒤로 버튼**
```gherkin
GIVEN 사용자가 학습 모드 선택 화면에 있을 때
WHEN "뒤로" 버튼을 클릭하면
THEN 단어 세트 상세 페이지로 돌아간다
AND URL이 `/wordsets/[id]`로 변경된다
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → URL 변경 확인
- 네비게이션 테스트: 브라우저 히스토리 확인

### AC-3: 플립 카드 인터랙션
**우선순위**: 높음 (필수)

**시나리오 3.1: 앞면 → 뒷면 플립**
```gherkin
GIVEN 사용자가 학습 화면에서 앞면(뜻) 카드를 보고 있을 때
WHEN 카드를 클릭하면
THEN 3D 플립 애니메이션이 실행된다 (rotateY 180deg, 600ms)
AND 뒷면(단어)이 표시된다
AND TTS 음성이 자동으로 재생된다
```

**검증 방법**:
- E2E 테스트: 카드 클릭 → 애니메이션 실행 → 뒷면 렌더링 확인
- 단위 테스트: FlipCard 컴포넌트 `isFlipped` 상태 전환 확인
- 스크린샷 테스트: 플립 전/후 UI 비교
- 성능 테스트: 60fps 유지 확인 (Chrome DevTools Performance)

**시나리오 3.2: 뒷면 → 앞면 플립**
```gherkin
GIVEN 사용자가 학습 화면에서 뒷면(단어) 카드를 보고 있을 때
WHEN 카드를 다시 클릭하면
THEN 역방향 3D 플립 애니메이션이 실행된다 (rotateY 0deg, 600ms)
AND 앞면(뜻)이 표시된다
```

**검증 방법**:
- E2E 테스트: 카드 클릭 → 애니메이션 실행 → 앞면 렌더링 확인
- 단위 테스트: FlipCard 컴포넌트 토글 동작 확인

**시나리오 3.3: 플립 애니메이션 성능**
```gherkin
GIVEN 사용자가 학습 화면에 있을 때
WHEN 카드를 10회 연속 클릭하면
THEN 모든 플립 애니메이션이 60fps를 유지한다
AND 애니메이션 지연이 없다 (smooth transition)
```

**검증 방법**:
- 성능 테스트: Chrome DevTools Performance 프레임레이트 측정
- 시각적 테스트: 느린 네트워크 환경 시뮬레이션

**시나리오 3.4: 카드 UI 스타일**
```gherkin
GIVEN 사용자가 학습 화면에 있을 때
WHEN 앞면(뜻) 카드를 확인하면
THEN 배경색이 `bg-primary-50`이다
AND 텍스트 크기가 `text-4xl`이다
AND "클릭하여 단어 보기" 가이드가 표시된다

WHEN 뒷면(단어) 카드를 확인하면
THEN 배경색이 `bg-white`이다
AND 텍스트 크기가 `text-5xl`이다
```

**검증 방법**:
- 단위 테스트: FlipCard 컴포넌트 스타일 클래스 확인
- 스크린샷 테스트: 앞면/뒷면 UI 비교

### AC-4: TTS 음성 재생
**우선순위**: 높음 (필수)

**시나리오 4.1: 자동 음성 재생**
```gherkin
GIVEN 사용자가 앞면(뜻) 카드를 보고 있을 때
WHEN 카드를 클릭하여 뒷면(단어)으로 플립하면
THEN 단어가 음성으로 자동 재생된다 (Web Speech API)
AND 스피커 아이콘에 애니메이션이 적용된다 (재생 중 표시)
AND 재생 완료 후 아이콘 애니메이션이 중지된다
```

**검증 방법**:
- 브라우저 수동 테스트: 카드 플립 → 음성 재생 확인
- 단위 테스트: useSpeech 훅 `speak` 함수 호출 확인
- Mock 테스트: SpeechSynthesisUtterance 생성 및 `speechSynthesis.speak` 호출 확인

**시나리오 4.2: 수동 음성 재생**
```gherkin
GIVEN 사용자가 학습 화면에 있을 때
WHEN "🔊 음성 듣기" 버튼을 클릭하면
THEN 현재 단어가 음성으로 재생된다
AND 버튼 텍스트가 "🔊 재생 중..."으로 변경된다
AND 재생 완료 후 "🔊 음성 듣기"로 복원된다
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → 버튼 텍스트 변경 확인
- 단위 테스트: useSpeech 훅 `isSpeaking` 상태 전환 확인

**시나리오 4.3: 앞면에서 음성 듣기**
```gherkin
GIVEN 사용자가 앞면(뜻) 카드를 보고 있을 때
WHEN "🔊 음성 듣기" 버튼을 클릭하면
THEN 단어를 미리 들을 수 있다 (뒷면 전환 없이)
AND 카드는 앞면(뜻) 상태로 유지된다
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → 카드 상태 확인 (플립 안 함)
- 단위 테스트: onSpeak 핸들러 독립적 실행 확인

**시나리오 4.4: TTS 미지원 브라우저**
```gherkin
GIVEN 사용자가 TTS를 지원하지 않는 브라우저를 사용할 때
WHEN 학습 화면에 접근하면
THEN "이 브라우저는 음성 재생을 지원하지 않습니다" 메시지가 표시된다
AND 음성 버튼이 비활성화 상태이다
```

**검증 방법**:
- Mock 테스트: `window.speechSynthesis` undefined 시뮬레이션
- 단위 테스트: useSpeech 훅 `isSupported: false` 확인

**시나리오 4.5: TTS 음성 설정**
```gherkin
GIVEN 시스템이 TTS 음성을 재생할 때
WHEN 음성 파라미터를 확인하면
THEN 언어가 'en-US'이다
AND rate(속도)가 0.9이다 (약간 느리게)
AND pitch(음높이)가 1.0이다 (기본값)
```

**검증 방법**:
- 단위 테스트: SpeechSynthesisUtterance 파라미터 확인
- 브라우저 수동 테스트: 음성 속도 및 음높이 확인

### AC-5: 네비게이션
**우선순위**: 높음 (필수)

**시나리오 5.1: 다음 카드 네비게이션**
```gherkin
GIVEN 사용자가 첫 번째 카드를 보고 있을 때
WHEN "다음" 버튼을 클릭하면
THEN 두 번째 카드의 앞면(뜻)이 표시된다
AND 진행률이 "2 / 5"로 업데이트된다
AND 진행률 바가 40% (2/5)로 업데이트된다
AND "이전" 버튼이 활성화된다
AND 세션 스토리지의 `currentIndex`가 1로 업데이트된다
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → 카드 전환 → 진행률 확인
- 단위 테스트: useLearnSession 훅 `next()` 함수 테스트
- 세션 스토리지 테스트: `currentIndex` 업데이트 확인

**시나리오 5.2: 이전 카드 네비게이션**
```gherkin
GIVEN 사용자가 세 번째 카드를 보고 있을 때
WHEN "이전" 버튼을 클릭하면
THEN 두 번째 카드의 앞면(뜻)이 표시된다
AND 진행률이 "2 / 5"로 업데이트된다
AND 진행률 바가 40% (2/5)로 업데이트된다
```

**검증 방법**:
- E2E 테스트: 버튼 클릭 → 카드 전환 → 진행률 확인
- 단위 테스트: useLearnSession 훅 `previous()` 함수 테스트

**시나리오 5.3: 첫 카드에서 이전 버튼 비활성화**
```gherkin
GIVEN 사용자가 첫 번째 카드를 보고 있을 때
WHEN 화면을 확인하면
THEN "이전" 버튼이 비활성화 (`disabled`) 상태이다
AND 버튼 스타일이 회색으로 표시된다 (opacity: 0.5)
```

**검증 방법**:
- E2E 테스트: 첫 카드에서 "이전" 버튼 `disabled` 속성 확인
- Accessibility 테스트: aria-disabled="true" 확인

**시나리오 5.4: 마지막 카드에서 학습 완료 버튼 표시**
```gherkin
GIVEN 사용자가 마지막 카드를 보고 있을 때
WHEN 화면을 확인하면
THEN "다음" 버튼 대신 "학습 완료" 버튼이 표시된다
AND 버튼 스타일이 primary variant이다 (bg-primary-600)
```

**검증 방법**:
- E2E 테스트: 마지막 카드에서 "학습 완료" 버튼 렌더링 확인
- 단위 테스트: LearnNavigation 컴포넌트 조건부 렌더링 확인

**시나리오 5.5: 네비게이션 시 플립 상태 초기화**
```gherkin
GIVEN 사용자가 뒷면(단어) 카드를 보고 있을 때
WHEN "다음" 또는 "이전" 버튼을 클릭하면
THEN 다음/이전 카드가 앞면(뜻)으로 표시된다
AND 플립 상태가 초기화된다 (isFlipped: false)
```

**검증 방법**:
- E2E 테스트: 뒷면에서 네비게이션 → 앞면 렌더링 확인
- 단위 테스트: useLearnSession 훅 네비게이션 시 `isFlipped` 초기화 확인

### AC-6: 진행률 표시
**우선순위**: 중간 (필수)

**시나리오 6.1: 진행률 텍스트 표시**
```gherkin
GIVEN 사용자가 학습 화면에 있을 때
WHEN 현재 5번째 카드를 보고 있으면
THEN 진행률 텍스트가 "5 / 20"으로 표시된다
AND 헤더 영역에 위치한다
```

**검증 방법**:
- E2E 테스트: 진행률 텍스트 렌더링 확인
- 단위 테스트: LearnProgress 컴포넌트 텍스트 포맷 확인

**시나리오 6.2: 진행률 바 애니메이션**
```gherkin
GIVEN 사용자가 첫 번째 카드를 보고 있을 때
WHEN "다음" 버튼을 클릭하면
THEN 진행률 바가 20%에서 40%로 스무스하게 증가한다
AND 트랜지션이 300ms 동안 적용된다
```

**검증 방법**:
- E2E 테스트: 진행률 바 width 속성 변화 확인
- 스크린샷 테스트: 진행률 바 애니메이션 전/후 비교
- 성능 테스트: 트랜지션 duration 확인

**시나리오 6.3: 진행률 바 스타일**
```gherkin
GIVEN 사용자가 학습 화면에 있을 때
WHEN 진행률 바를 확인하면
THEN 전체 바 배경색이 `bg-neutral-200`이다
AND 진행된 부분 색상이 `bg-primary-600`이다
AND 높이가 `h-1`이다 (4px)
```

**검증 방법**:
- 단위 테스트: LearnProgress 컴포넌트 스타일 클래스 확인
- 스크린샷 테스트: 진행률 바 UI 비교

### AC-7: 학습 완료
**우선순위**: 높음 (필수)

**시나리오 7.1: 학습 완료 화면 표시**
```gherkin
GIVEN 사용자가 마지막 카드를 보고 있을 때
WHEN "학습 완료" 버튼을 클릭하면
```
